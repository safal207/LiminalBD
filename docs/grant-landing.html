A replayable memory and evidence layer for adaptive systems.

LiminalDB is the storage layer of the Liminal Stack: an open database for systems that need memory, replay, auditability, and adaptive behavior in one place.

Why this layer matters

ProblemTraditional storage systems keep records, but often lose the reasoning trail of how an adaptive system changed.
ConsequenceTeams rebuild replay, event sourcing, evidence, and protocol access on top of generic stores.
ContributionLiminalDB gives the stack a memory and evidence primitive built for replay and open access.

The grant case

Open access

LiminalDB exposes protocol and SDK surfaces instead of hiding state behind one vendor boundary.

Replayable history

Mirror Timeline makes change history persistent and inspectable, which is critical for adaptive systems.

Stack fit

It acts as the memory and evidence layer beneath routing and runtime components in the Liminal Stack.

What reviewers should remember

  • Adaptive systems need more than storage; they need replayable operational memory.
  • Open protocol access matters for digital commons infrastructure.
  • LiminalDB exists to make system change persistent, inspectable, and reusable.

Best next milestone
