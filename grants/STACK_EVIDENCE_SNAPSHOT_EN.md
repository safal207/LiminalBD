# Liminal Stack — short evidence blurb for reviewers (EN)

We submit a three-repository stack where each layer is backed by measured behaviour, not only architectural claims. **DAO_lim** demonstrates end-to-end resilience under backend failure: traffic fails over to a healthy upstream while requests remain successful, with routing decisions exposed through explain output and circuit state on unhealthy backends. **GardenLiminal** publishes Linux microbenchmarks for control-plane paths (manifest parsing and lifecycle event serialization), showing that YAML/JSON overhead is not a dominant bottleneck for the runtime layer. **LiminalBD** publishes a first single-node WebSocket baseline (live LQL round-trip plus ingest+batch with an LQL probe), with reproducible commands and explicit caveats in [`docs/BENCHMARKS.md`](../docs/BENCHMARKS.md) and a pointer from [`README.md`](../README.md). Together this reads as open infrastructure with verifiable properties rather than a concept deck. We position the current numbers honestly as a developer baseline on a single machine, distinct from modelled comparisons and from future reviewer-grade expansion (soak, replay, multi-node, CI performance gates). The next milestone is a fuller benchmark package; for submission, the stack already presents a credible first evidence layer.
