# Liminal Stack — краткий текст для заявки / cover note (RU)

Мы подаёмся с трёхрепозиторным стеком, где каждый слой уже подкреплён измеримым поведением, а не только архитектурными заявлениями. **DAO_lim** показывает живой end-to-end сценарий устойчивости: при сбое primary трафик переключается на fallback, запросы остаются успешными, а решение маршрутизации прозрачно через explain и состояние circuit breaker. **GardenLiminal** даёт микробенчмарки control-plane путей на Linux (разбор манифестов и сериализация lifecycle-событий), демонстрируя, что накладные расходы YAML/JSON слоя не выглядят узким местом для runtime. **LiminalBD** публикует первый measured baseline для single-node WebSocket пути: live LQL round-trip и ingest+batch сценарий с воспроизводимыми командами и явными caveats в [`docs/BENCHMARKS.md`](../docs/BENCHMARKS.md), плюс ссылка из [`README.md`](../README.md). Вместе это выглядит как инфраструктурный стек с доказуемыми свойствами (failover/explainability, дешёвый control plane, live latency/throughput baseline), а не как набор концептов. Мы честно позиционируем текущие цифры как developer baseline на одной машине и отделяем их от modelled сравнений и будущих reviewer-grade пакетов (soak, replay, multi-node, CI perf gates). Следующий шаг после подачи — расширить измерения до полноценного benchmark package, но для грантовой истории «инженерная зрелость + измеримость» уже закрыта на уровне первого сильного доказательного слоя.
